# 招新活动管理模块 产品需求文档 (PRD)

## 1. 模块概述

### 1.1 背景
招新活动管理模块用于管理学生会面向新生的招新活动，包括招新活动的创建、编辑、发布、报名审核等全流程管理。

### 1.2 目标
- 提供独立的招新活动管理功能
- 支持招新活动的全生命周期管理
- 提供报名数据统计和导出功能
- 支持面试安排和录用管理

### 1.3 与报名管理的区别
| 维度 | 招新活动管理 | 报名管理 |
|------|-------------|----------|
| 目标用户 | 新生/新成员 | 全体学生 |
| 活动性质 | 长期招新活动 | 单次活动报名 |
| 流程复杂度 | 高（含面试、录用） | 低（仅报名） |
| 数据字段 | 多（含籍贯、志愿等） | 少（基础信息） |

## 6. 状态流转

```
                    ┌─────────────┐
                    │   待审核    │
                    └──────┬──────┘
                           │
           ┌───────────────┼───────────────┐
           │               │               │
           ▼               ▼               ▼
    ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
    │    已通过   │ │   已拒绝    │ │   面试中    │
    └─────────────┘ └─────────────┘ └──────┬──────┘
                                           │
                              ┌────────────┴────────────┐
                              │                         │
                              ▼                         ▼
                       ┌─────────────┐          ┌─────────────┐
                       │    已录用   │          │   未通过    │
                       └─────────────┘          └─────────────┘
```

状态说明：
- PENDING: 待审核
- APPROVED: 已通过（直接通过，无需面试）
- REJECTED: 已拒绝
- INTERVIEW: 面试中
- HIRED: 已录用
- FAILED: 面试未通过

## 8. 交互细节

### 8.1 创建招新活动
1. 点击"创建招新"按钮
2. 填写招新信息表单
3. 点击"保存"保存为草稿，或"发布招新"直接发布

### 8.2 审核报名
1. 在报名列表点击"查看"打开详情
2. 查看报名者完整信息
3. 点击"通过"直接通过，或"安排面试"进入面试流程，或"拒绝"拒绝申请

### 8.3 安排面试
1. 点击"安排面试"按钮
2. 选择面试时间和输入面试地点
3. 系统自动发送通知（后续可接入短信/邮件）
4. 报名状态变为"面试中"

### 8.4 导出数据
1. 点击"导出数据"按钮
2. 生成CSV文件下载
3. 包含所有报名信息
